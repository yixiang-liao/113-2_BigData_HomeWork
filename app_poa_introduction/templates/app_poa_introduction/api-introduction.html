甚麼是Restful API

你有沒有呼叫過Google, Facebook, Instagram, Twitter的API? 你看到的API，呼叫方式，資料回傳的格式等，基本上都遵循某種風格，稱RESTful API Representational State Transfer，簡稱REST，它是一種網路API架構的一種風格(不是一種標準)。


RESTful API

RESTful API是一種client - server 架構，可以將複雜的資料、複雜的演算法、重要的資源獨立(分離)出來，透過API介面讓別人使用。

  • 將Deep Learning Keras模型寫成RESTful API，開放給他人呼叫使用，別人可以用手機程式、網頁、或是其他程式，寫一個有用的整合系統，呼叫你的API結果。
  • Django有REST framework套件可用，但這裡我們不用套件，我們用最簡單的方式完成任務。

What is Restful API?

Did you call Google, Facebook, Instagram, or Twitter API before? Yes, they are Restful APIs.

Representational State Transfer, or REST in short, is an architectural style that defines web service to allow internet users to access your API.

Representational state transfer (REST) is a software architectural style that defines a set of constraints to be used for creating Web services. Web services that conform to the REST architectural style, called RESTful Web services, provide interoperability between computer systems on the Internet. RESTful Web services allow the requesting systems to access and manipulate textual representations of Web resources by using a uniform and predefined set of stateless operations. Other kinds of Web services, such as SOAP Web services, expose their own arbitrary sets of operations.
